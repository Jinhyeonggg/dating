# 코드 리뷰

$ARGUMENTS (미입력 시 최근 변경된 파일 전체) 에 대해 다음 기준으로 리뷰한다.

## 리뷰 항목

### 1. Claude API 사용 효율
- system prompt가 너무 길지 않은가? (300토큰 이하 권장)
- 대화 히스토리가 불필요하게 누적되지 않는가?
- 20턴 제한이 올바르게 구현되었는가?
- 스트리밍이 필요한 곳에 스트리밍이 사용되는가?

### 2. 보안
- API 키가 코드에 하드코딩되지 않았는가?
- 사용자 입력이 Claude system prompt에 직접 삽입되지 않는가? (프롬프트 인젝션)
- `.env` 파일이 .gitignore에 포함되는가?

### 3. 페르소나 격리
- 클론 A의 컨텍스트가 클론 B에게 누출되지 않는가?
- 각 시뮬레이션 실행이 독립적인가?

### 4. Python 코드 품질
- 타입 힌트가 모든 함수 시그니처에 있는가?
- async/await가 일관되게 사용되는가?
- Pydantic 모델로 입출력이 검증되는가?

### 5. TypeScript 코드 품질
- `any` 타입이 없는가?
- 에러 상태와 로딩 상태가 UI에서 처리되는가?

### 6. UI 안정성 (레이아웃 점프 금지)
- 같은 리스트/그리드 안의 카드 높이·너비가 콘텐츠 길이에 따라 달라지지 않는가? (`h-full` + placeholder + `min-h`)
- 탭·아코디언 전환 시 패널 너비가 변하지 않는가? (`grid-cols-[Xrem_minmax(0,1fr)]`)
- 스크롤바가 나타날 때 가로가 흔들리지 않는가? (`scrollbar-gutter: stable`)
- textarea에 `resize-none`이 있는가? 자동 리사이즈가 걸려 있지 않은가?
- 스켈레톤 크기가 실제 콘텐츠와 일치하는가?
- 이미지·아바타에 `width/height` 또는 `aspect-ratio`가 명시되어 있는가?
